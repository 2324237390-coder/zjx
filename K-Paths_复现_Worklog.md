# K-Paths 复现 Worklog

本次工作验证：

1. K-Paths 是否能通过医学知识图谱路径提高大模型预测效果。
2. K-Paths 是否适合用于探索知识图谱中的弱关联或未知关联，并据此提出科研假设。

官方项目：

- 代码：<https://github.com/rsinghlab/K-Paths>

## 3. 实验设置

- 任务：药物-疾病治疗关系分类
- 数据集：PharmacotherapyDB，252条测试样本
- 模型：
  - Qwen2.5-14B-Instruct
  - Qwen2.5-32B-Instruct
- Base：仅输入问题、候选标签和模型自身知识
- K-Paths：在 Base 基础上增加最多10条、最长3跳的知识图谱路径
- 其他 prompt 和推理参数保持一致

标签包括：

| 标签 | 含义 |
|---|---|
| Disease-modifying | 改变疾病基础或下游生物学 |
| Palliates | 缓解症状但不改变疾病进程 |
| Non-indication | 不改变疾病，也不用于缓解症状 |

### Base 与 K-Paths 的区别

K-Paths 不是一个新的大模型，而是给大模型检索知识图谱证据的方法。

Base 只向模型提出问题：

```text
Drug A 对 Disease C 是否具有治疗作用？
```

模型只能依赖自身参数中已有的医学知识。

K-Paths 会先在知识图谱中查找两个实体之间的间接路径：

```text
Drug A -> binds -> Gene B
Gene B -> is associated with -> Disease C
```

然后把问题和路径一起输入同一个模型。

| Base | K-Paths |
|---|---|
| 只有问题和候选答案 | 问题、候选答案和知识图谱路径 |
| 依赖模型自身知识 | 同时使用外部图谱证据 |
| 不提供关系证据链 | 可以展示支持预测的多跳路径 |

因此，本次对照实验实际测量的是：同一个模型获得知识图谱路径后，能否比只依赖自身知识
做出更好的判断。

## 4. 实验结果

| 模型 | 设置 | Accuracy | Macro-F1 | Kappa |
|---|---|---:|---:|---:|
| Qwen2.5-14B | Base | 60.71 | 60.98 | 42.23 |
| Qwen2.5-14B | K-Paths | **65.48** | **63.02** | **47.56** |
| 14B 增益 | | **+4.76** | **+2.04** | **+5.33** |
| Qwen2.5-32B | Base | 65.48 | 65.08 | 47.78 |
| Qwen2.5-32B | K-Paths | **70.63** | **65.70** | **52.83** |
| 32B 增益 | | **+5.16** | **+0.62** | **+5.05** |

Qwen2.5-32B + K-Paths 与论文结果接近：

| 来源 | Accuracy | Macro-F1 | Kappa |
|---|---:|---:|---:|
| 论文 | 71.83 | 67.83 | 55.10 |
| 本次复现 | 70.63 | 65.70 | 52.83 |
| 差距 | -1.20 | -2.13 | -2.27 |

## 5. 结果分析

### 整体表现

- 两个模型加入 K-Paths 后，三个指标都提高。
- Accuracy 和 Kappa 均提高约5个百分点。
- 14B 的 Macro-F1 提高2.04，说明较小模型能从外部图知识中获得更多补充。
- 32B 的 Macro-F1 只提高0.62，但 Accuracy 提高5.16。

因此，K-Paths 在当前任务中有效，但提升幅度属于中等，不是所有指标都大幅提升。

### 类别影响

| 模型 | 设置 | Disease-modifying F1 | Palliates F1 | Non-indication F1 |
|---|---|---:|---:|---:|
| 14B | Base | 56.25 | 61.03 | 65.65 |
| 14B | K-Paths | **71.88** | **66.67** | 50.51 |
| 32B | Base | 65.62 | 67.01 | 62.61 |
| 32B | K-Paths | **77.69** | **72.73** | 46.67 |

K-Paths 明显增强了治疗关系识别，但降低了 `Non-indication` 表现。

可能原因是药物、基因、疾病之间的生物学关联路径容易被模型进一步解释为治疗证据。
这说明图路径质量和关系语义会直接影响模型，相关关系不能自动等同于治疗或因果关系。

252条样本上的配对 McNemar 检验约为 `p=0.17`。当前结果方向一致且接近论文，
但仍需在更多数据和任务上验证稳定性。

## 6. 是否适合我们的医学项目

### 对科研假设提出任务的适用性

本项目的目标是探索知识图谱中的弱关联或未知关联，并基于这些关联提出可验证的科研假设。
K-Paths 可以用于该任务，但不能单独完成整个流程。

原始 K-Paths 的前提是先给定一对查询实体，例如“Drug A”和“Disease C”，然后：

1. 查找两个实体之间多条关系序列不同的间接路径。
2. 将路径作为该未知关系的支持证据。
3. 让模型判断两个实体之间可能存在什么关系。

因此，K-Paths 适合承担“候选关联证据检索”环节，但它本身不会自动遍历整个知识图谱，
找出所有值得研究的未知实体对。

针对本项目，建议采用：

```text
候选未知实体对生成
-> K-Paths 提取多条间接证据
-> 计算关联可信度和新颖性
-> 大模型生成科研假设
-> 文献与专家验证
```

候选未知实体对可以来自：

- 图中没有直接边，但存在2至3跳连接的实体对
- 具有多条独立路径支持的实体对
- 关系类型不同但机制方向一致的实体对
- 图嵌入或链接预测模型给出的高分缺失边
- 图中有机制证据、但文献中共同出现较少的实体对

大模型生成的科研假设应至少包含：

- 推测的新关系
- 支持该关系的 K-Paths 证据
- 可能的生物学机制
- 与已有知识相比的新颖性
- 可能的反向证据
- 可验证的实验或数据分析方案

需要注意：默认 K-Paths 偏向最短路径和已有高连接节点，可能遗漏更弱、更长或更罕见的
关联。用于科研假设发现时，应增加路径新颖性、关系罕见度、证据来源和边置信度评分，
不能只按最短路径排序。

总体判断：

**K-Paths 可以作为科研假设提出系统的证据检索模块，但需要结合候选实体对发现、
新颖性排序和科学验证模块，才能形成完整策略。**

## 7. 接入所需的输入输出

### 必要输入

1. 医学知识图谱三元组：

```text
源实体 - 关系 - 目标实体
```

2. 实体ID、名称和类型，如药物、疾病、基因。
3. 能转换为自然语言的关系名称。
4. 待预测的实体对。
5. 明确的任务标签和训练/测试划分。

业务数据中的实体必须和知识图谱实体完成可靠对齐。

### 框架输出

每个实体对会得到：

- `all_paths`：节点ID和关系ID组成的结构化路径
- `all_paths_str`：带实体名称和关系名称的路径
- `path_str`：可直接输入大模型的自然语言证据

例如：

```text
Drug A binds Gene B
and Gene B is associated with Disease C
```

最终输出为关系预测标签和对应知识图谱证据链。

## 8. 最终结论

本次复现表明，K-Paths 能够提高 Qwen2.5-14B 和32B 在药物重定位任务上的整体表现，
结果方向与论文一致，32B结果接近论文报告值。

因此，该框架具备迁移到我们医学项目的价值，尤其适合药物、疾病、基因等实体之间的
关系预测和证据增强。

建议先选择一个明确的医学实体关系任务进行小规模试点，同时运行 Base 与 K-Paths，
重点评估：

- 总体指标是否稳定提升
- 是否存在类别性能恶化
- 路径是否具有合理医学含义
- 专家是否认可模型使用的证据链

满足以上条件后，再扩大到正式医学业务。
