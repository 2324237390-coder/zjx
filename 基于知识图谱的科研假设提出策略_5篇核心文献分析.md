# **1. Literature Based Discovery (LBD): Towards Hypothesis Generation and Knowledge Discovery in Biomedical Text Mining**

| **类型** | 综述文章 / 理论入口                                    |
|----------|--------------------------------------------------------|
| **作者** | Balu Bhasuran, Gurusamy Murugesan, Jeyakumar Natarajan |
| **年份** | 2023 arXiv；期刊参考为 Medinformatics, 2026            |
| **链接** | [<u>arXiv 页面</u>](https://arxiv.org/abs/2310.03766)  |
| **DOI**  | 10.48550/arXiv.2310.03766                              |

## **1.1 为什么适合你的任务**

这篇文献是“弱关联发现”的理论入口。你的任务强调探索知识图谱中的弱关联或未知关联，而 LBD 的核心正是从文献中发现尚未被直接连接的医学概念关系。它把“科研假设生成”看作一种从已有知识中连接分离领域的过程。

## **1.2 核心思想**

LBD 最经典的是 ABC 模式：如果 A 与 B 有关系，B 与 C 有关系，但 A 与 C 尚未被直接研究，那么 A-C 可以作为候选科研假设。

在医学中，A 可以是疾病，B 可以是基因、机制、症状、通路或表型，C 可以是药物、干预、环境因素或诊断指标。

- Open discovery：只给定 A，系统寻找可能的中介 B，再返回潜在 C，用于生成新假设。

- Closed discovery：给定 A 和 C，系统寻找解释二者潜在关系的中介 B，用于解释或验证一个已有猜想。

- 弱关联的本质：A-C 之间没有显式强连接，但通过多个 B 形成间接证据链。

## **1.3 可转化成你的策略**

在你的系统中，可以把 LBD 的 ABC 思路改造成“KG-ABC 弱关联搜索”：

输入：目标疾病 A；输出：若干 A-B-C 或 A-B1-B2-C 路径；再由 LLM 将路径转成研究问题、机制假设和验证实验。

| **步骤**     | **实现含义**                                                                    |
|--------------|---------------------------------------------------------------------------------|
| A 节点选择   | 疾病、临床问题、核心表型或研究主题                                              |
| B 节点发现   | 与 A 强相关但不一定直接构成研究结论的机制节点，如基因、通路、炎症机制、代谢过程 |
| C 节点扩展   | 与 B 相连但与 A 直接连接较弱或缺失的药物、干预、biomarker、实验模型             |
| A-C 假设生成 | 把 A-B-C 路径转化为“C 是否通过 B 影响 A？”这样的可检验假设                      |

## **1.4 局限与注意事项**

- ABC 模式容易产生大量候选，需要过滤机制，否则弱关联会变成噪声关联。

- 共现不等于机制关系，需要区分文献共现、统计相关和生物因果机制。

- 在医学场景中，必须保留证据来源、时间戳、实体标准化信息和上下文限制。

# **2. PrimeKG: Building a Knowledge Graph to Enable Precision Medicine**

| **类型**       | 资源论文 / 医学知识图谱底座                                              |
|----------------|--------------------------------------------------------------------------|
| **作者**       | Payal Chandak et al.                                                     |
| **年份与期刊** | Scientific Data, 2023                                                    |
| **链接**       | [<u>Nature 页面</u>](https://www.nature.com/articles/s41597-023-01960-3) |
| **DOI**        | 10.1038/s41597-023-01960-3                                               |

## **2.1 为什么适合你的任务**

如果说 LBD 告诉你“如何发现弱关联”，PrimeKG 则告诉你“医学知识图谱应该长什么样”。它不是一个假设生成算法，而是一个可作为医学图谱底座的 schema 和资源范例。

## **2.2 核心内容**

PrimeKG 整合多个高质量医学资源，覆盖疾病、药物、基因/蛋白、通路、表型、解剖、暴露因素、副作用、适应症、禁忌症、off-label use 等医学实体与关系。它的价值在于把分散在不同数据库和临床文本中的医学知识统一到一个多模态图谱中。

- 节点层面：疾病、药物、蛋白/基因、生物过程、分子功能、细胞组分、表型、解剖结构、暴露因素、副作用等。

- 边层面：drug-disease、disease-protein、protein-pathway、disease-phenotype、drug-side effect、drug-indication/contraindication/off-label use 等。

- 证据层面：强调疾病实体标准化、跨资源整合、临床文本描述与结构化图谱结合。

## **2.3 可转化成你的策略**

PrimeKG 对你最有价值的是 schema 设计。你可以先不从零构建完整医学 KG，而是参考它定义第一版任务图谱：

| **模块**     | **建议采用的图谱字段**                                                                   |
|--------------|------------------------------------------------------------------------------------------|
| 实体         | Disease, Drug, Gene/Protein, Pathway, Phenotype, Side Effect, Clinical Trial, Paper      |
| 关系         | treats, targets, associated_with, participates_in, causes, contraindicated_for, supports |
| 边属性       | evidence_source, confidence, direction, context, timestamp, supporting_sentence          |
| 假设生成输入 | 目标疾病子图 + 候选缺失边 + 证据路径 + 文献证据                                          |

## **2.4 局限与注意事项**

- PrimeKG 解决的是图谱底座问题，不直接解决“如何产生假设”。

- 不同数据源之间的关系强度、证据质量和时间更新频率不一致，需要单独建模。

- 若直接用于发现未知关联，需要额外叠加 link prediction、path reasoning 或 LLM hypothesis generation。

# **3. DREAMwalk: Biomedical Knowledge Graph Learning for Drug Repurposing by Extending Guilt-by-Association to Multiple Layers**

| **类型**       | 方法论文 / 未知关联预测                                                  |
|----------------|--------------------------------------------------------------------------|
| **作者**       | Dongmin Bang, Sangsoo Lim, Sangseon Lee, Sun Kim                         |
| **年份与期刊** | Nature Communications, 2023                                              |
| **链接**       | [<u>Nature 页面</u>](https://www.nature.com/articles/s41467-023-39301-y) |
| **DOI**        | 10.1038/s41467-023-39301-y                                               |

## **3.1 为什么适合你的任务**

DREAMwalk 是“从医学 KG 中预测未知关联”的代表方法。它关注 drug-disease association prediction，也就是在已有 drug-gene-disease 图谱上预测潜在的新药物适应症。这非常适合作为你系统中的候选假设生成器。

## **3.2 核心思想**

DREAMwalk 把 guilt-by-association 从单层网络扩展到 drug-gene-disease 多层图谱。传统 guilt-by-association 的含义是：功能相似或邻域相似的实体可能具有相似功能。该文进一步利用语义信息引导随机游走，让药物和疾病都能被映射到统一的 embedding 空间，从而进行 drug-disease 关联预测。

- 输入：包含 drug、gene、disease 多层关系的 biomedical KG。

- 方法：semantic information-guided random walk 生成节点序列，再学习 embedding。

- 输出：潜在 drug-disease association 排名。

- 验证：报告相对多种 SOTA link prediction 模型的性能提升，并展示 breast carcinoma 与 Alzheimer’s disease case study。

## **3.3 可转化成你的策略**

你可以把 DREAMwalk 对应为 Minerva 中的“未知边预测器”：

| **系统位置**         | **设计**                                                                |
|----------------------|-------------------------------------------------------------------------|
| GraphExplorer        | 对目标疾病的候选 drug-disease、gene-disease、pathway-disease 边进行预测 |
| Hypothesis Generator | 把高分未知边转写成“某药物/靶点是否通过某机制影响某疾病”的假设           |
| Evidence Retriever   | 寻找预测边周围的药物-基因、基因-疾病、通路-疾病证据                     |
| SuperJudge           | 结合预测分数、证据链、可验证性和新颖性对候选假设排序                    |

## **3.4 局限与注意事项**

- embedding 预测通常解释性不足，需要结合路径解释方法补足机制链。

- 高分未知边不等于高质量科研假设，还需要考虑临床可行性、毒性、副作用、研究空白程度和可验证实验。

- 适用于药物重定位场景，但若你的研究对象是机制发现或 biomarker 发现，需要扩展目标关系类型。

# **4. Explainable Drug Repurposing via Path-Based Knowledge Graph Completion / XG4Repo**

| **类型**       | 方法论文 / 路径化可解释推理                                              |
|----------------|--------------------------------------------------------------------------|
| **作者**       | A. Jiménez et al.                                                        |
| **年份与期刊** | Scientific Reports, 2024                                                 |
| **链接**       | [<u>Nature 页面</u>](https://www.nature.com/articles/s41598-024-67163-x) |
| **DOI**        | 10.1038/s41598-024-67163-x                                               |

## **4.1 为什么适合你的任务**

这篇文献直接对应“弱关联/未知关联如何被解释”。在科研假设生成中，预测一条未知边还不够，必须解释这条边为什么可能成立。XG4Repo 的价值在于把 drug repurposing 形式化为 KG completion，并要求输出能被专家理解的路径证据。

## **4.2 核心思想**

该文把 drug repurposing 看成 query answering：给定 compound 和 treats 关系，预测可能的 disease。与纯 embedding 方法不同，path-based 方法会从药物出发，在异构图中寻找通向候选疾病的多跳路径。路径可以包含 target、gene、pathway、side effect、anatomy 等中间节点。

- Path：具体节点和边组成的一条证据链。

- Metapath：节点类型和关系类型组成的路径模板，也可理解为规则。

- 解释目标：让专家知道“为什么这个药物可能治疗这个疾病”。

- 工程价值：候选结果可以以排序列表 + 自然语言解释的形式输出。

## **4.3 可转化成你的策略**

这篇文献对你的最大启发是：不要让系统只输出一个弱关联，而要输出“弱关联 + 支撑路径 + 路径解释”。

| **输出字段**       | **示例**                                                                     |
|--------------------|------------------------------------------------------------------------------|
| candidate_relation | Drug X may affect Disease Y                                                  |
| evidence_path      | Drug X → targets Protein A → regulates Pathway B → associated_with Disease Y |
| metapath_type      | Drug-targets-Protein-participates_in-Pathway-associated_with-Disease         |
| path_score         | 路径可靠性、长度、边置信度、多源证据数量                                     |
| hypothesis_text    | Drug X 是否可通过调控 Protein A / Pathway B 改善 Disease Y？                 |

## **4.4 局限与注意事项**

- 路径短不一定代表机制真实，路径长也不一定无用，需要区分“解释性”和“因果性”。

- 路径搜索空间很大，必须控制路径类型、长度、节点度数和证据质量。

- 需要防止高连接度 hub 节点造成虚假路径，例如炎症、免疫、TP53 等高频节点可能连接过多疾病。

# **5. HypoChainer: A Collaborative System Combining LLMs and Knowledge Graphs for Hypothesis-Driven Scientific Discovery**

| **类型** | 系统论文 / LLM + KG + 人类专家协同假设生成                  |
|----------|-------------------------------------------------------------|
| **作者** | Haoran Jiang, Shaohan Shi, Yunjie Yao, Chang Jiang, Quan Li |
| **年份** | 2025 arXiv 预印本                                           |
| **链接** | [<u>arXiv 页面</u>](https://arxiv.org/abs/2507.17209)       |
| **DOI**  | 10.48550/arXiv.2507.17209                                   |

## **5.1 为什么适合你的任务**

HypoChainer 是这5篇中最接近“系统设计”的一篇。它关注的问题是：GNN/KG 可以产生大量预测，但人工筛选不可扩展；LLM 可以辅助生成和筛选假设，但容易幻觉。因此系统把 KG、LLM、可视分析和专家反馈组合起来形成 hypothesis-driven scientific discovery 流程。

## **5.2 核心流程**

- Contextual exploration：专家用 RAG、可视化和解释工具探索大量 GNN 预测，找到研究焦点。

- Hypothesis chain formation：围绕候选预测检查 KG 关系和语义相关实体，逐步形成假设链。

- Validation prioritization：根据 KG-supported evidence 过滤和排序候选假设，选出优先实验验证对象。

## **5.3 可转化成你的策略**

HypoChainer 可以作为 Minerva 的 GraphExplorer + Scientific Thinker 交互流程参考。一个可迁移的设计是：

| **HypoChainer 阶段**       | **Minerva 对应模块**                                                    |
|----------------------------|-------------------------------------------------------------------------|
| Contextual exploration     | GraphExplorer：发现弱关联、聚类候选预测、展示证据子图                   |
| Hypothesis chain formation | Scientific Thinker：把图谱路径转化为机制链和研究假设                    |
| Validation prioritization  | Graph-enhanced SuperJudge：按证据一致性、新颖性、可验证性和潜在影响排序 |

## **5.4 局限与注意事项**

- 系统依赖人类专家参与，不是完全自动化闭环。

- 更偏交互式可视分析框架，模型训练与 reward learning 部分不是重点。

- 对 Minerva 来说，应把它与核心参考文献中的 Scientific Judge / Scientific Thinker 结合：KG 负责 grounding，Judge 负责价值判断，Thinker 负责假设表达与方案生成。

# **7. 面向你任务的策略设计建议**

建议把“基于知识图谱的科研假设提出策略”拆成五个连续模块，而不是只做单一 link prediction。

## **7.1 模块一：弱关联/未知关联发现**

- ABC 路径发现：寻找 A-B-C 或 A-B1-B2-C 间接路径，适合从疾病出发开放式发现。

- KG completion / link prediction：预测图谱中缺失的 drug-disease、gene-disease、pathway-disease、biomarker-disease 关系。

- 路径模板搜索：定义医学上合理的 metapath，例如 Drug→Protein→Pathway→Disease。

- 局部稀疏/空白发现：寻找高潜力但研究密度较低的节点、边或子图。

## **7.2 模块二：证据链构建**

每个候选未知关联必须附带证据链，而不是只给一个分数。建议至少输出：

- 支撑路径：从候选起点到终点的多跳路径。

- 路径类型：metapath / rule。

- 证据来源：数据库、文献、临床试验、实验数据。

- 边置信度：关系抽取置信度、数据库可信度、文献数量、时间新旧。

- 反证信息：是否存在禁忌症、负相关、失败临床试验或矛盾文献。

## **7.3 模块三：LLM 假设生成**

LLM 的任务不是凭空生成 idea，而是把“候选弱关联 + 图谱证据链”转化成结构化科研假设。建议输出模板如下：

| **字段**              | **说明**                                         |
|-----------------------|--------------------------------------------------|
| Research question     | 用问题形式表达待验证关系                         |
| Hypothesis            | 明确提出机制性假设                               |
| Mechanistic rationale | 解释图谱路径如何支持该假设                       |
| Evidence chain        | 列出关键节点、边、证据来源                       |
| Novelty signal        | 说明该关系为何属于弱关联/未知关联/低研究密度方向 |
| Validation experiment | 给出可执行验证方案                               |
| Failure mode          | 说明可能失败的原因和反证条件                     |

## **7.4 模块四：Graph-enhanced SuperJudge 排序**

对候选假设进行 pairwise comparison 或多维评分。建议评价维度包括：

- Novelty：是否不是显而易见的已知强连接。

- Evidence consistency：图谱证据是否方向一致、来源可靠。

- Mechanistic plausibility：路径是否具有合理生物机制。

- Verifiability：是否能设计明确实验或数据分析验证。

- Impact：若成立是否有医学或科学价值。

- Risk：是否存在高幻觉风险、路径 hub 偏差、反证或安全性问题。

## **7.5 模块五：验证与反馈**

- Temporal holdout：用 T 年前的图谱生成候选，检查 T 年后是否出现新论文、新数据库边或临床试验。

- 专家评审：让医学专家评价机制合理性和验证价值。

- 下游实验反馈：把实验结果或文献后验验证结果反哺 SuperJudge。

- 模型能力画像：记录不同模型/agent 在不同假设生成任务上的表现。

# **8. 推荐阅读顺序与产出物**

- 第一步读 LBD：写清楚弱关联发现的基本范式和 ABC 模式。

- 第二步读 PrimeKG：整理医学 KG 的 schema、节点/边类型和证据字段。

- 第三步读 DREAMwalk：总结未知 drug-disease 边预测策略。

- 第四步读 XG4Repo：总结如何把预测结果变成路径解释。

- 第五步读 HypoChainer：总结如何把图谱预测、LLM 推理、专家反馈组合成 hypothesis chain。

建议最终交付三个产出物：1）文献综述；2）基于 KG 的假设生成策略图；3）第一版 GraphExplorer 数据结构与 pipeline 设计。

# **9. 可直接用于调研报告的总述段落**

针对“如何探索知识图谱中的弱关联或未知关联，并基于这些关联提出有效科研假设”这一任务，现有文献可以被组织为一条从知识发现到假设生成的技术链条。LBD 提供了从分离文献集合中发现潜在 A-C 关系的经典 ABC 范式；PrimeKG 提供了面向精准医学的多尺度知识图谱底座；DREAMwalk 说明了如何通过多层 guilt-by-association 和图表示学习预测未知 drug-disease 关联；XG4Repo 强调通过 path-based knowledge graph completion 为预测结果提供可解释机制路径；HypoChainer 则进一步展示了如何将 KG、LLM、人类专家和可视分析整合为 hypothesis chain 生成与验证流程。因此，本项目可以将图谱视为候选假设生成器、证据链提供者和假设评价上下文，再与 SuperScientist / Scientific Judge 框架结合，形成“弱关联发现—机制路径构建—假设生成—图谱增强评价—实验验证反馈”的闭环。
