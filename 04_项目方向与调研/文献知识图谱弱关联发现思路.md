# 文献知识图谱弱关联发现思路

## 1. 目标

本工作希望从大量论文构成的知识图谱中，找出“目前没有被直接研究，但通过其他论文和研究发现看起来可能有联系”的组合。

这些组合称为“弱关联”。弱关联不是已经证明的结论，而是值得继续查证的研究线索。

## 2. 当前知识图谱中有什么

当前知识图谱可以简单理解为一张“论文关系网”，其中包含：

- `Paper`：一篇论文。
- `Author`：论文的作者，一篇论文可以连接多位作者。
- `Finding`：论文得出的一条研究发现，负责记录这条发现来自哪篇论文。
- `Entity`：某条研究发现中的具体对象，例如药物、指标、人群、模型、行为、现象或任务。它可以在发现中充当主体或客体。
- `CanonicalConcept`：`Entity` 对应的标准概念，用于把同一概念的不同写法统一起来。
- `Context`：某条研究发现成立时的限定条件。第一版弱关联发现中暂时不使用。
- `MethodologyInstance`：某篇论文实际使用的具体方法。
- `MethodologyType`：具体方法所属的方法大类。
- `Topic`：论文研究的主题。

主要连接方式为：

```text
Paper -AUTHORED_BY-> Author
Paper -REPORTS-> Finding
Finding -HAS_SUBJECT-> Entity（主体）
Finding -HAS_OBJECT-> Entity（客体）
Entity -具体关系-> Entity（例如 HIGHER_THAN）
Entity -MAPS_TO-> CanonicalConcept
Finding -UNDER_CONTEXT-> Context（第一版暂时忽略）
Paper -> MethodologyInstance -> MethodologyType
Paper -> Topic
```

简单理解：论文由作者撰写，论文报告了研究发现，一条发现指出了主体和客体，并记录两个对象之间的具体关系。每个研究对象还可以对应一个标准概念。

例如，图中某条 `Finding` 可以表示：

```text
Entity A -HIGHER_THAN-> Entity B
```

`Finding` 让我们知道这条关系是哪篇 `Paper` 报告的，`MAPS_TO` 则让我们知道这两个 `Entity` 对应哪些标准概念。

### 2.1 CanonicalConcept是干什么的

不同论文可能使用不同说法表示同一个概念，例如：

```text
抑郁
抑郁症
depression
```

这些 `Entity` 的写法不同，但可能都映射到同一个 `CanonicalConcept`。因此，`Entity` 保留论文中的具体说法，`CanonicalConcept` 则用来识别它们实际上是同一个概念。

这对弱关联发现很重要：系统不应该把“抑郁—抑郁症”当成一个新的弱关联。第一版可以用 `CanonicalConcept` 进行合并和去重，同时保留原始 `Entity` 作为论文证据。

## 3. 三类弱关联

### 3.1 Entity—Entity：两个研究对象可能有关

例如，已有论文分别发现：

```text
睡眠不足 -> 炎症升高
炎症升高 -> 记忆下降
```

图谱中即使没有“睡眠不足—记忆下降”的直接连接，它们也通过“炎症升高”间接连在一起。

可以由此提出问题：

> 睡眠不足会不会通过炎症反应影响记忆？

这类弱关联的意义是发现新的对象关系，以及可能存在的中间原因或作用路径。

在实际计算时，可以先利用 `CanonicalConcept` 将同义的 `Entity` 合并后再寻找弱关联，输出结果时再展示原始 `Entity`、`Finding` 和 `Paper` 作为依据。

### 3.2 Entity—MethodologyType：一种方法可能用于新的研究对象

例如：

```text
方法A经常用于研究焦虑
睡眠问题与焦虑密切相关
但很少有研究使用方法A研究睡眠问题
```

可以由此提出问题：

> 能否把研究焦虑的方法A，用来研究睡眠问题？

这类弱关联的意义是把已有方法应用到新的研究对象上，也就是“换一种方法研究一个问题”。

### 3.3 Topic—Entity：某个研究主题可能忽略了一个对象

例如：

```text
“老年认知”主题经常研究记忆和睡眠
炎症指标又分别与记忆和睡眠有关
但该主题下很少直接讨论炎症指标
```

可以由此提出问题：

> 在“老年认知”研究中，炎症指标是不是一个尚未被充分关注的对象？

这类弱关联的意义是发现某个研究领域中可能被忽略的对象，帮助拓展研究方向。

## 4. 三类弱关联的简单区别

| 弱关联类型 | 要回答的问题 | 可能形成的想法 |
| --- | --- | --- |
| Entity—Entity | 它们会不会有关？ | A可能影响B，或A与B可能存在共同原因 |
| Entity—MethodologyType | 能不能用这种方法研究它？ | 可以使用方法C研究A |
| Topic—Entity | 这个领域是不是忽略了它？ | 主题D可以进一步关注A |

## 5. 发现弱关联的基本流程

1. 将论文、作者、研究发现、研究对象、标准概念、方法和主题连成知识图谱。
2. 利用 `CanonicalConcept` 将同一概念的不同 `Entity` 写法对齐。
3. 在图谱中找出没有直接连线的两个节点。
4. 检查它们之间是否存在较强的间接联系，例如共享相关论文、研究对象、发现、方法或主题。
5. 为这些没有直接连线的组合计算分数并排序。
6. 展示排名靠前的组合，同时给出它们在图谱中的间接路径。
7. 由研究人员阅读相关论文，判断该组合是否合理、是否新颖、是否值得研究。

## 6. DREAMwalk可以起到什么作用

DREAMwalk原本是在医学知识图谱中学习药物、基因和疾病之间的间接联系。对它进行改造后，可以让它在文献知识图谱中学习 `Paper`、`Finding`、`Entity`、`CanonicalConcept`、`MethodologyType` 和 `Topic` 之间的间接联系。`Author` 可以保留在图谱中，但第一版可以先不把作者关系作为弱关联判断的主要依据。

第一版改造可以先完成以下任务：

- 保留DREAMwalk在多种节点和关系上寻找路径的思路。
- 将药物、基因和疾病节点替换为文献图谱中的节点。
- 对没有直接关系的候选组合计算间接联系强度。
- 输出排名靠前的弱关联及其中间路径。

第一版可以暂时不判断“它们一定有关”或“一定无关”，而是只做排序：间接联系越强，越值得优先人工查看。

## 7. 预期输出

弱关联生成器最终应该输出一份候选清单，每条至少包含：

- 弱关联两端的对象。
- 弱关联类型。
- 推荐分数。
- 连接两端的中间路径。
- 支持该路径的论文。
- 一句简单的候选研究问题。

示例：

```text
候选对象：睡眠不足—记忆下降
类型：Entity—Entity
中间线索：睡眠不足—炎症升高—记忆下降
候选问题：睡眠不足是否可能通过炎症反应影响记忆？
```

## 8. 需要注意的问题

- 弱关联只是线索，不是证明它们真的有关。
- 应先利用 `CanonicalConcept` 识别同义概念，避免把同一个概念的不同写法当成弱关联。
- 间接路径多，不代表研究想法一定合理。
- 需要检查支持路径的原始论文，避免知识抽取错误。
- 需要确认候选关系是否早已被其他论文直接研究。
- 最终的科研假设需要由研究人员结合专业知识进行判断。
